# 01. 프로젝트 개요

## 프로젝트 소개

**The Promise**는 교회 공간 대여를 온라인으로 신청하고 관리하는 시스템입니다.
기존에 수기로 관리하던 공간 예약을 데이터화하여 관리 효율을 높이는 것이 첫 번째 목표입니다.

---

## 개발 단계

### Phase 1 — 현재 목표: 수기 → 온라인 데이터화

핵심 기능만 만들어서 빠르게 실제로 써봅니다.

| 기능 | 설명 |
|------|------|
| 예약 신청 | 신청자 정보 + 공간 + 시간 입력 후 제출 |
| 예약 조회 | 이름/연락처로 본인 예약 확인 |
| 자동 승인/거절 | 시간 중복 없으면 자동 승인, 있으면 자동 거절 |
| 관리자 대시보드 | 전체 예약 달력 + 상세 조회, 예약 취소 |

### Phase 2 — 알파/베타 테스트 후 개선

실제 사용자 피드백을 받아서 수정합니다. 구체적인 항목은 테스트 후 결정.

### Phase 3 — 기능 확장

- 공간 CRUD (관리자가 직접 공간 추가/수정/삭제)
- 예약 필터/검색 고도화
- 팀 목록 사전 정의
- 반복 예약 (매주 등)
- 기타 피드백 반영

---

## 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| 프론트엔드 | React + TypeScript (Vite) | 타입 안정성, 팀 기술 스택에 맞음 |
| 백엔드 | Python + Django + DRF | 빠른 개발 속도, 내장 관리자 페이지, ORM |
| 데이터베이스 | PostgreSQL | 안정적이고 확장성 있음, Django와 궁합 좋음 |
| 비동기 처리 | Celery + Redis | 알림 발송 비동기 처리, 예약 처리 직렬화 |
| 패키지 관리 | pnpm workspaces | JS/TS 패키지 모노레포 관리 |

> **DRF**: Django REST Framework. Django에서 API(백엔드가 프론트에게 데이터를 주는 통로)를 쉽게 만들 수 있게 해주는 라이브러리입니다.

> **Celery**: Django와 함께 쓰는 비동기 작업 처리 라이브러리. Redis를 메시지 브로커로 사용합니다.

> **pnpm workspaces**: JS/TS 패키지만 관리합니다. `apps/api/`(Python)는 pnpm 범위 밖이며 폴더로만 존재합니다.

## 대상 사용자

| 사용자 | 역할 |
|--------|------|
| 일반 신청자 | 공간 예약 신청, 본인 예약 조회 |
| 관리자 | 전체 예약 조회, 예약 취소 |

---

## 예약 가능 범위

- **오픈 건물**: 3개 건물 (추후 확장 예정)
- **예약 시간 단위**: 30분
- **예약 가능 시간**: 24시간
- **예약 방식**: 단건 예약 (반복 예약은 Phase 3)
- **중복 처리**: 같은 공간, 같은 시간대 예약 자동 거절
  - 판단 기준: `신청 시작시간 < 기존 종료시간 AND 신청 종료시간 > 기존 시작시간`
