# Phase 1.5 백엔드 Plan

## 목표

예약 신청/취소 시 신청자에게 알림을 발송합니다.
Celery + Redis를 활용해 API 응답과 알림 발송을 분리(비동기)합니다.

---

## 알림 수단 비교

### 이메일 (Gmail SMTP)

| 항목 | 내용 |
|------|------|
| 비용 | 무료 (구글 계정 필요) |
| 발송 한도 | 하루 500건 |
| 구현 난이도 | 낮음 |
| 필요 작업 | `Reservation` 모델에 `applicant_email` 필드 추가 |
| 단점 | 신청 폼에 이메일 입력 필드 추가 필요. 전화번호 기반 서비스와 맞지 않음 |

```
pip install 추가 없음 (Django 내장 send_mail 사용)
환경변수: EMAIL_HOST_USER, EMAIL_HOST_PASSWORD
```

---

### SMS (솔라피)

| 항목 | 내용 |
|------|------|
| 비용 | 가입 시 무료 크레딧 제공 / 이후 단문(SMS) 약 8~10원/건 |
| 발송 한도 | 제한 없음 (과금) |
| 구현 난이도 | 낮음 |
| 필요 작업 | 솔라피 가입 + API 키 발급 |
| 단점 | 소액이지만 과금 발생. 사업자 인증 필요할 수 있음 |

```
pip install solapi
환경변수: SOLAPI_API_KEY, SOLAPI_API_SECRET, SOLAPI_SENDER_PHONE
```

> 참고: https://developers.solapi.com

---

### 카카오 알림톡

| 항목 | 내용 |
|------|------|
| 비용 | 건당 약 8원 |
| 발송 한도 | 제한 없음 (과금) |
| 구현 난이도 | 높음 |
| 필요 작업 | 카카오 비즈니스 채널 개설 + 알림톡 템플릿 심사 (수일~수주 소요) |
| 단점 | 채널 개설 및 템플릿 심사 절차가 복잡하고 시간이 걸림. 교회 단체 채널 필요 |

```
pip install 없음 (직접 REST API 호출)
환경변수: KAKAO_API_KEY, KAKAO_SENDER_KEY, KAKAO_TEMPLATE_CODE
```

> 참고: 카카오 비즈니스 채널 → 알림톡 발신 프로필 등록 → 템플릿 등록 및 심사 순서로 진행

## 구현 예정 흐름 (수단 결정 후 작성)

```
예약 신청 API 응답 (201)
    ↓ (비동기)
Celery task → Redis 큐 → 알림 발송
```

- 발송 성공/실패 여부는 별도 로깅 처리 예정
- 발송 실패 시 최대 3회 재시도 (`max_retries=3`)
